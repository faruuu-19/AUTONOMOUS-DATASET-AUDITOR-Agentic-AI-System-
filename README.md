# Autonomous Dataset Auditor

An agentic AI system for auditing tabular datasets before model training.

This project checks common data risks such as leakage, train-test contamination, class imbalance, spurious correlations, and low-utility features. It includes both a command-line interface and a Streamlit web app for interactive analysis.

## What It Does

- Audits CSV datasets before modeling
- Selects and runs specialized detectors
- Produces a readiness score and final verdict
- Generates actionable recommendations
- Exports JSON reports and findings CSVs
- Includes synthetic test datasets and validation scripts

## Audit Areas

- Data leakage
- Train-test contamination
- Bias and class imbalance
- Spurious correlations
- Feature utility and redundancy

## Project Structure

```text
agent/         Agent memory, planning, critic, goals, and strategy logic
tools/         Specialized dataset risk detectors
data/          Example datasets
test_data/     Generated datasets and test outputs
reports/       Saved audit reports
main.py        Command-line entry point
app.py         Streamlit web app
auditor.py     Core autonomous auditor
```

## Requirements

- Python 3.10+
- pip

## Installation

```bash
pip install -r requirements.txt
```

## Run The CLI

Audit a single dataset:

```bash
python main.py --dataset data/test_dataset.csv --target target
```

Save the report to a custom path:

```bash
python main.py --dataset data/test_dataset.csv --target target --output reports/my_report.json
```

Export findings to CSV:

```bash
python main.py --dataset data/test_dataset.csv --target target --export-csv reports/findings.csv
```

Use train/test split files:

```bash
python main.py --train train.csv --test test.csv --target target
```

## Run The Web App

```bash
streamlit run app.py
```

Then upload a CSV file, choose the target column, and start the audit from the sidebar.

## Output

The system produces:

- A verdict: `READY`, `NEEDS ATTENTION`, or `NOT READY`
- A readiness score out of 100
- Critical issues, warnings, and informational findings
- Recommendations for cleanup before modeling
- JSON and CSV exports for reporting

## Example Reports

Generated reports are typically saved under `reports/`, including:

- `reports/audit_report.json`
- `reports/findings.csv`
- `reports/test_audit_report.json`

## Test Scripts

Run the end-to-end audit demo:

```bash
python test_full_audit.py
```

Run the broader autonomous behavior suite:

```bash
python test_autonomous_system.py
```

You can also run the focused detector and agent tests in the repository root, such as `test_leakage.py`, `test_bias.py`, and `test_planner.py`.

## Notes

- Input datasets are expected to be CSV files.
- The target column must exist in the dataset.
- Some files in `data/`, `test_data/`, and `reports/` are example artifacts generated by local runs.
